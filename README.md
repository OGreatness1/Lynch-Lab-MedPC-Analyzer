# Lynch Lab MedPC Analyzer 🧬
**Version 5.0 (Modular Production Release)**



## Overview
This is a robust, modular data pipeline designed specifically for MedPC behavioral data generated by the Lynch Lab. It automatically parses 15+ years of historical protocol variants into clean Excel files and generates pattern-detecting visual analytics.

## Folder Structure
* `app.py`: The Streamlit web interface.
* `parser.py`: The resilient parsing engine that handles messy `.txt` output.
* `analyzer.py`: The core data-wrangling engine for computing arrays and scalars.
* `plotter.py`: The visualization generation module.
* `config.py`: The master dictionary containing 30+ MSN mappings.
* `utils.py`: ID normalization and gender extraction helpers.

## How to Run Locally
1. Ensure you have Python 3.9+ installed.
2. Install dependencies: `pip install streamlit pandas numpy matplotlib seaborn openpyxl`
3. Start the server: `streamlit run app.py`

## Adding a New Experiment Protocol
If a student programs a new experiment in MedPC:
1. Open `config.py`.
2. Add the `MSN` name to `DEFAULT_MSN_PATTERNS`.
3. If it uses standard variables (I, R, etc.), assign it to `map_rat_fr` in `DEFAULT_VARIABLE_MAPPINGS`.
4. The system will automatically accommodate the new format.

## Troubleshooting
* **"No data found"**: Verify the subject ID matches exactly (the system automatically corrects `O` vs `0` errors).
* **Missing Sessions**: Check the `00_Skipped_Sessions_Log.xlsx` file generated in the ZIP download. This details exactly which text block failed and why.
